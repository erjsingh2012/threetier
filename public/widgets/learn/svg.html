SVG — 80% Explained (Interactive Guide)

A compact, hands-on reference that covers the most commonly used parts of SVG: shapes, paths, styling, viewBox, transforms, grouping, gradients, masks, filters, text, symbols, and simple animation. Try the live examples and tweak attributes in the code boxes.

Quick reference (cheat sheet)

Coordinate system: x to the right, y down. Use viewBox="minX minY width height" to scale.
Common elements: <rect> <circle> <ellipse> <line> <polyline> <polygon> <path>
Styling: fill, stroke, stroke-width, opacity, CSS works directly on SVG shapes.

Basic shapes

Rectangles, circles, ellipses, lines, polylines, polygons are the building blocks. Use them for UI, icons, charts.

<rect x="8" y="8" width="120" height="56" rx="10" fill="#0b78d1" stroke="#0b78d1"/> <circle cx="200" cy="36" r="28" /> <ellipse cx="320" cy="36" rx="46" ry="22" /> <line x1="8" y1="96" x2="140" y2="96" /> <polyline points="180,76 220,96 260,56" /> <polygon points="360,96 400,56 440,96" />

Paths (the swiss army knife)

Paths describe arbitrary shapes using commands (M, L, H, V, C, S, Q, T, A, Z). Use paths for icons, complex shapes and strokes.

<path d="M160 60 C 180 0, 260 0, 280 60 S 340 120, 380 60" /> <path d="M20 100 Q 80 40, 140 100" /> <path d="M320 40 A 40 40 0 0 1 420 40" />
Quick path tips
  • Lowercase commands use relative coordinates; uppercase use absolute.
  • Use stroke-linecap and stroke-linejoin to control stroke ends and corners.
  • To draw smooth joins, try using S and T which continue control points.

viewBox & coordinate-system scaling

viewBox="minX minY width height" defines the internal SVG coordinate system. It allows the SVG to scale responsively. preserveAspectRatio controls alignment and scaling behavior.

viewBox="0 0 200 100" maps SVG coords 0..200 horizontally and 0..100 vertically into the rendered area
<svg viewBox="0 0 200 100" width="320" height="160"> <circle cx="50" cy="50" r="30"/> <rect x="120" y="20" width="60" height="60"/> </svg> // preserveAspectRatio example <svg viewBox="0 0 100 100" preserveAspectRatio="xMidYMid slice"> ... </svg>

Transforms

Use transform attributes: translate(x,y), rotate(angle [cx cy]), scale(sx [,sy]), skewX(), skewY(), matrix(). Can be applied to individual shapes or groups (<g>).

<g transform="translate(20,14)"> <rect x="0" y="0" width="80" height="40"/> <rect transform="rotate(15 140 20)" x="100" y="0" width="80" height="40"/> </g>

Fill, stroke & paint server (gradients/patterns)

The two most important properties: fill and stroke. Use opacity, fill-opacity, stroke-opacity, stroke-dasharray.

<linearGradient id="g1" x1="0" x2="1"> <stop offset="0%" stop-color="#0b78d1"/> </linearGradient> &lt;rect fill="url(#g1)" /> &lt;pattern id="p1" width="20" height="20" patternUnits="userSpaceOnUse"> ...
When to use patterns vs gradients

Use gradients for smooth color transitions, patterns for repeated textures or tiled graphics. SVG patterns can include nested shapes and even images (via <image>).

Grouping, defs, symbols and <use>

Group with <g> to apply transforms or styles to multiple elements. Put reusable markup in <symbol> or <defs> and instantiate with <use href="#id" x=".." y=".."/> to avoid duplication.

<symbol id="btn" viewBox="0 0 100 40"> ... </symbol> <use href="#btn" x="10" y="10"/> // <use> copies markup; it can be styled by the referencing element.

Clipping & masking

Use <clipPath> to limit drawing to a shape (binary). Use <mask> to create soft-edge transparency using luminance/alpha of the mask content.

<clipPath id="c1"><circle cx="70" cy="70" r="60"/></clipPath> <image href="..." clip-path="url(#c1)"/> <mask id="m1"> ... </mask> // mask uses luminance/alpha to control final transparency.

Note: external images in <image> require correct CORS headers to be used with canvas or exported.

Filters (blur, drop shadow, color effects)

Filters are powerful but can be expensive. Use <filter> with primitives like feGaussianBlur, feOffset, feColorMatrix, feBlend to compose effects.

<filter id="f1"> <feGaussianBlur in="SourceAlpha" stdDeviation="4"/> <feOffset dx="6" dy="6"/> <feMerge> ... </feMerge> </filter> <rect filter="url(#f1)"/>

Text

Text in SVG supports <text>, <tspan> and textPath for following curves. Use text-anchor and dominant-baseline for alignment. Fonts are loaded via CSS and inherit like HTML.

<text x="10" y="20">Simple text</text> <textPath href="#curve">...</textPath>

Accessibility: include role="img" and aria-label when SVG conveys meaning. Provide fallback text with <title> and <desc> inside the SVG.

Animation basics

You can animate SVG with SMIL (<animate>), CSS animations/transitions, or JavaScript. SMIL animation still works in most modern browsers; for complex sequences consider JS libraries (GreenSock, anime.js).

<circle cx="40" cy="40" r="18"> <animate attributeName="cx" from="40" to="220" dur="1.2s" repeatCount="indefinite"/> </circle> // CSS anim example: // svg circle { transition: transform .3s } // circle:hover { transform: translateX(8px) scale(1.02); }

Accessibility, performance & exporting

  • Accessibility: add <title> and <desc> tags, role="img" and aria-label attributes, and ensure focus order if interactive.
  • Performance: avoid overly complex filters, reduce node count (use <use>), rasterize complex shadows in bitmap form when possible, prefer CSS for hover effects.
  • Exporting: external images need CORS to render in canvas. To export SVG as PNG, inline fonts or use web fonts available to the rendering environment.

Helpful tips & pitfalls

  1. Remember that y axis grows downward by default — flip with transform or viewBox if you prefer math coords.
  2. Units: numbers are user units (usually px). Use percentages in attributes that accept them (like patternUnits or viewBox mapping).
  3. Use patternUnits="userSpaceOnUse" when you want patterns to be fixed-size in screen pixels.
  4. When debugging sizing problems, draw a rect at 0,0 with width/height equal to viewBox to inspect how coordinates map.

Advanced topics (filters, accessibility, JS animation)

This filters section shows composing multiple primitives for rich effects. Use filter region expansion when blur exceeds bounds.

Accessibility examples

Include <title> and <desc>, role and aria-labels. Make interactive elements keyboard-focusable (tabindex) and use <rect> or <button> with role to indicate interactivity.

<svg role="img" aria-label="..."> <title>...</title> <desc>...</desc> <rect tabindex="0" role="button"/> </svg>

JS-driven animation sequence (control + pause)

Use JS for precise control over animations (start, stop, seek). Example below animates a circle along a path and exposes controls.

// JS moves element along path using getPointAtLength const path = document.getElementById('animPath'); const dot = document.getElementById('animDot'); let raf, t0; function step(ts){ if(!t0) t0 = ts; const progress = Math.min((ts - t0)/2000, 1); const p = path.getPointAtLength(progress * path.getTotalLength()); dot.setAttribute('cx', p.x); dot.setAttribute('cy', p.y); if(progress < 1) raf = requestAnimationFrame(step); else t0 = null; } // call requestAnimationFrame(step) to start
